## Sloving MNIST
- 분류 모델에서의 카테고리는 class라고 명칭
- data point = samples

- 신경망의 핵심 구성 요소는 레이어
  - 레이어는 입력된 데이터로부터 표현을 추출
  - 훈련이 진행되면 될수록 유용한 표현
- 네트워크를 훈련할 준비
  - 손실함수
  - 최적화
  - 훈련 및 테스트 모니터링 할 지표
 
- Dataset and Dataloader -> 데이터 공급 파이프라인 정의
- Dataset에서 미니배치를 스트리밍 방식으로 가져오기
- transforms.ToTensor() -> [0,1] 범위로 스케일링 
- 데이터는 텐서로 저장되어있음
- 텐서는 데이터를 담는 컨테이너
  #### - 텐서의 축(axis)의 개수 (또는 차원수)를 랭크라고 한다
  - 하나의 숫자만을 담고 있는 텐서는 사칼

#### -> train_images 는 8비트 정수로 이루어진 3차원 텐서이다
#### -> 28*28 정수로 이루어진 행렬이 60000개 모여있는 배열
#### -> 각 행렬은 하나의 흑백 이미지
#### -> 각 원소의 값은 0에서 255사이의 값가

- DataLoader가 데이터를 미니배치 단위로 나누어 전달
- 이때 데이터로더는 설정된 배치크기 DataLoader(batch_size =128)에 맞는 묶음을 계속 제공

### 데이터의 종류
1) 벡터 데이터 -> (샘플, 특징) 형태의 2차원 텐서
2) 시계열 데이터 -> (샘플, 시간단계, 특징) -> 3차원 텐서 ex. 주식가격 데이터셋
3) 이미지 데이터 -> (샘플, 채널, 높이, 너비) -> 4차원 텐서 , 이미지 데이터엔 텐서 존재 -> 흑백 or 컬러
   -> 우린 채널을 먼저 쓸것임
4) 비디오 데이터 -> (샘플, 프레임, 채널, 높이, 너비) -> 5차원 텐서
   -> 60초짜리 144X256 초단 4프레임 샘플링 240프레임 영상 4개 -> (4,240, 144,256,3)

### 텐서 연산
- Linear 레이어
  
  
   -> self.fc1 = nn.Linear(28*28, 512)
   
   -> x = self.fc1(x)
   
   -> x = F.relu(x)

   -> relu(dot(W,input)+b)

- 3가지 텐서 연산
  1) 입력 텐서와 W간의 dot.product (내적)
  2) 결과 텐서와 b를 더하는 연산
  3) relu연산

### 브로드 캐스팅
- 두 텐서의 shape이 다를때 덧셈은 어떻게 처리됨?
- > 각각의 shape를 맞춰줌
  1) 작은 텐서에 축 추가해서 큰 텐서와 차원을 맞춤
  2) 추가된 축을 따라 작은 텐서를 반복해서 큰 텐서의 shape맞춤
     > x.shape (32,10) y.shape(10,)  -> y.shape(1,10)  -> y.shape(32.10)
